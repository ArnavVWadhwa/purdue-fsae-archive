Purdue Formula SAE : 8/25/2019 - PF19 GP Track (Aero)

Created by Unknown User (agarw162), last modified on Oct 15, 2020

Summary

We measured down-force with linear potentiometers attached to all four shocks on the car. These measure spring displacement and by using the spring constant of the spring you can calculate the lift coefficient produced by the Aerodynamic package. Before you take the data from each run you must create limiting factors to generate a Steady-State environment on the track so the values of coefficient aren't being influenced by other variable. This was done using the ECU to create a rev limiter at the top of second gear. By doing so, you essentially limit the max velocity of the car at a reasonable number and therefore can pull data from that value. If you didn't create a Steady-State environment and rather drove like normally in a race (called Transient) you would  create a negative pitching moment due to the acceleration of the car and get high lift coefficient values. This was compared to the CFD by taking the geometry from the 2019 season and running the geometry in a Spallart-Allmaras turbulence model environment to check correlation which will then be tied with Wind Tunnel testing results. However, because of the noise of the data, this has a large uncertainty which is why the data used a filter to average out the values over a long duration and remove any noise created by bumpy track, wind, etc. 

Methodology

Before you take the raw data of the MoTec log files you must first meet certain conditions and apply some filters as stated above. All the raw MoTec Log Files are linked below alongside the excel spreadsheet to take the results. 

Steps to taking data values

  • Ensure that the data is a Steady-State simulation by checking the indication of the gear at the bottom of the MoTec log file alongside general trends of graph. It is fairly easily to spot if the run was either Steady-State or Transient. 
  • Apply low pass filter to some of the data to filter out the noise 
    • Be very careful when applying a low pass filter since it filters out data above a given set range of Hz so if most of your true data-set lies outside the range you defined then you won't be capturing the data
  • Look for regions of reliability steady (low oscillations) of RPM and where RPM is at the highest possible
  • Narrow down on the region where Steering Angle is also steady
  • Take the snippet of data just before the driver applies the brakes since that would create a positive pitching moment
  • Look at the time range of the data that you are taking 
    • Make sure that the time range isn't extremely low (i.e 0.3 seconds where everything is steady is pretty pointless) 
  • Check that the shock pots are also relatively steady for the given time range 
  • Repeat for all sets of data with the same filtering process and keeping the time range as consistent as possible between data pulls

 Steps for Calculations

  • Spring Rate (lb/in)
    • 350
  • Air Density (kg/m^3)
    • 1.1839
    • Depends on altitude and air temperature
  • Calculate delta compression/expansion for each shock by subtracting maximum and minimum to receive value in mm
  • Convert delta of shocks into inches by dividing by 25.4 
  • Get Force in lbs by multiplying delta by spring rate 
  • Add rear force terms and front force terms and convert to Newtons to calculate Cl 
  • Make sure to obtain vehicle speed for each case in m/s
