Purdue Formula SAE : Build Rework Objectives | PF22

Created by Skrzyszowski, Mateusz, last modified by Paradise, Hunter on Jul 16, 2023

These are the objectives with creating a fresh M1 Build project. The big picture idea with this project is to make it easier to navigate the M1 Tune interface and deepen our understanding with what is actually happening with our Build and Tune.


  1. Remove everything that is unnecessary in Build.
    1. This clutters up the tune and makes it more difficult to find the parameter you want to change or the channel you want to read.Â 
    2. Stuff like Air Conditioning, Boost, Alternative Fuel, etc. isn't used by us but still shows up in the Tune. These things need to be removed in Build completely.
  2. Remove Enumerations that may include things that were pulled out of the build.
    1. This is just to make it clear that options that existed for some parameters no longer exist
  3. Add control for everything that is not a part of the base GPRP (type of ECU we have) template.
    1. This includes things like DRS, Dyno Control (for the dyno spec of the project), etc.
    2. This also includes adding sensors that are wired to the ECU to build.
  4. Configure communication between nodes on the CAN bus.
    1. In order for the ECU to read sensors that are wired to the PDM or Dash, communication must be configured. This means opening messages, setting CAN message IDs, receiving messages, etc..
    2. CAN message IDs must be tracked so no two messages have the same ID. This is across all devices connected to the CAN bus.
  5. Edit build items that may be more useful for our Tune.
    1. By this, I mean things like changing the max amount of labels an axis can have in a table.
    2. Need to work with Carson on this to figure out what would make his life easier when tuning the engine.
