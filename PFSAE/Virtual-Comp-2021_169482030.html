PF20 vs PF22 for design presentation - ITS PF20

  • Overview:
    • PF22 Pros:
      • Better discussion of architecture decision via lap sim
      • Better discussion of "making most" out of covid times
    • PF22 Cons:
      • More work to provide car values
  •  Sus
    • PF22 Pros:
      • Better discussion of lap time simulation usage
    • PF22 cons:
      • Validation less direct, but still mostly applicable
      • Need to update various dynamic values + work out what springs the lighter car wants (bad, but also good)
      • Lose opportunity to have direct suspension curve checks and stuff
  • Chas
    • PF22 Pros
      • get feedback on design (basically the same tho so...)
      • subsystems have been analyzed more completely (pedalbox and steering)
    • PF22 cons
      • some more design work to do before (but maybe this is a pro?)
      • not sure if we can actually do this given I have submitted SES with the old car?
  • Pow
    • PF22 Pros
      • design feedback for next year
      • could corelate r6 confirmed models to wr450f model (WAVE)
      • engine selection / vehicle architecture good to showcase
    • PF22 cons
      • no dyno testing
      • no car testing
      • everything is theoretical, and we always got roasted for no dyno testing / validation in the past
  • Elec
    • PF22 Pros
      • More 'cutting edge' stuff to talk about
        • DRS
        • Single cylinder
      • It'd be nice to see if they like how we talk about above
    • PF22 cons
      • Most of the stuff isn't validated - at best by time presentation is submitted we will have tested servo but probably not even that so if they ask about how single cylinder build or DRS works in practice ... we don't know
  • Aero
    • PF22 Pros
      • Showing a package with better performance, hence more competition points
      • Has better Yaw sensitivity as compared to PF20 design
    • PF22 cons
      • Summer package was tailored to R6 intake 
        • Have some numbers with that with the PF22 chassis and removed intake/exhaust but no further analysis 
        • Could possibly use Raunaq's PF22 CAD that has even better numbers than summer baseline and for WR450 car but that only has numbers and no additional analysis
      • Don't have mounting/structures work done yet but that's can be done but will be pushing it alongside doing the report properly and finishing PF20
        • Could possibly delay PF20 finish deadline 
      • Don't have Aero maps for this package and this definitely won't happen by the design report deadline 
        • Can be added as an appendix and just state that we would have this analysis in the presentation 
    • Takeaway: 
      • We could possibly use the PF22 package but the presentation would have a lot of blank areas that would need to be filled in with the appendices in the actual presentation
