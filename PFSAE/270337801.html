Purdue Formula SAE : How to import part/assembly files from a local drive | Teamcenter

Created by Dien Quang Nguyen, last modified on Oct 06, 2023

POC: Dien Quang Nguyen 

Description: This page outlines the steps required to import a part/assembly file from a local drive to Teamcenter

Notes: 

Currently, the team uses Teamcenter 13 to organize all CAD files. This version, as offered by Purdue ECN is only compatible with (as far as we are aware) NX 1946/NX 1926. 

Any CAD files that have been opened or edited/saved in any later NX versions (NX 2007/2027 and later) can't be imported to Teamcenter 13.

This is current as of August 25th, 2023.

Step 0: Prerequisites 

This guide assumes that you already have Teamcenter set up according to team standards and have launched the instance of NX that is linked to Teamcenter.

Refer to the page below to set up Teamcenter and NX if you do not meet this prerequisite:

  • Teamcenter: How to Install

If you already have NX and Teamcenter set up, check that you have done the preparation below.


Step 1: Preparation

To import a CAD file into Teamcenter, the file must be saved as a part file (.prt). Note that both assembly files and component part files are saved as .prt file.

Additionally, CAD files shown as "Creo Versioned File" or similar will also work.

Correction, you can import STEP files directly into Teamcenter and NX, follow this tutorial to see how that works → Teamcenter: How to import purchased parts


Step 2: Launch the Teamcenter-instance of NX 

This can be done in the Teamcenter Rich Client. 

(Click the button circled in red)


Once launched, it should look something just like the regular NX with an additional Teamcenter tab on the left navigation bar. This means this NX instance is linked with Teamcenter, and that we can access/modify files stored in Teamcenter from this NX instance.


Step 3: Import settings

Navigate to "File" → "Import Assembly into Teamcenter..."


You should now see a pop-up window like below.

Set "Item Type" to be "Item", [will have more explanation on reasons for doing this in the future, for now just make sure it is set to "Item" ]


Before going further, I recommend changing the "Name and Attribute Conversion" settings as shown below.

This automatically sets the Item ID to be identical to the file name of your local parts. More explanation on why this is done will be given later.


Additionally, if you are importing a part from a past vehicle that will remain the same for the new vehicle (ex: a PF23 part file that will be on the PF24 car). You can select the "Conversion Rule" as "With Replace String", then modify the "String to Replace" and "Replacement String" categories to automatically update your Item ID.

There is the assumption that naming convention remain the same from one year to another, but replacement strings can be cleverly utilized even if this is not the case. In the example below, I have the settings to rename all "PF23" strings to "PF24".



Step 4: Select your desired files

Click on "Select Assemblies or Parts"

This can be used to import a single part file or an assembly file.

The option "Import from Folder" can be used if you are sure that all your desired files are already stored in a local folder.


Navigate to the location of your local files and select your desired files. Click OK once your files are selected.

For assembly files, you would need to import all component files to be able to properly import the assembly into Teamcenter (as shown in the example above - for the NARCO 17in rack assembly), unless it is your intention to import only the assembly file.


Your import window should look something like this.


Step 5: Set item attributes

The IDs were automatically set, but you can manually modify them all by resetting the attributes. 

First select the files for which you want to modify the attributes (ID, Name, etc.). Click on the "Reset attributes" button.


Now all your fields should be reset.

Note that double-clicking on a field will automatically assign a value for the Item ID, Item Name, and Item Revision fields.

To manually input the Item ID, click once on the field and type in your Item ID.

The Item ID is what will be used to search for your files once imported into Teamcenter, so make sure you set the Item ID properly, following the team's CAD naming convention.


Step 6: Set the destination folder

Once your Item ID, Item Name, and Item Revision fields are all set, scroll over to the "Default Override" category and set the "Destination Folder".


Highlight all your files/file and right-click. Choose "Edit" to set the destination folder for all of them at once.


In the pop-up window, click on the "Folder Select" button. You should see a new pop-up window.


For the "Look In" dropdown, select "Home". This is your Teamcenter home directory, unique and only visible to each individual Teamcenter user. 

This means only you can see your folders, but all CAD (and other) files once added to Teamcenter can be searched and accessed by anyone. Access, permission, and check-in/check-out guidelines are (will be) covered in another Confluence page.


From here, select the folder you wish to import your files into (double-click on the folder to select). Click "OK" to confirm your selection, and click "OK" again.


Destination folders for your imported files should now be set.


Step 7: Import your file

Now that your attributes and destination folders are set, click "OK" to finish the import process.


You should see an Information window showing the "Teamcenter Integration Import Log"


If you also see the lines "Import operation completed successfully" in the log, it means you have succeeded!


Now, you can navigate to the set destination folder in Teamcenter to find your files.


Launching the assembly in NX from Teamcenter should also launch all component files.
