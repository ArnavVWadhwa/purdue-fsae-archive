Purdue Formula SAE : DAQ PF24 | Good rules of thumb

Task at hand: precision analog measurements with minimal noise & drift

Design rules: basically whatever you can get for the cheapest spec the fabhouse has to offer

  • Trace width: 6mil
  • Via drill: 0.3mm
  • Tented vias
  • Four layers when possible


GOOD HARDWARE DESIGN GUIDELINES:

  1. add redundancies.
    1. Expose pins for comms lines in case on-board components fail
    2. Just make sure stubs aren't too long and EMI is low. Solder pads work as well.  
      1. Ex) if onboard CAN transceivers fail, we can hook up external off-the-shelf modules to exposed CAN pins as backup
  2. Add test points!!! Kinda goes in hand with first rule
  3. Separate pin headers should match with 2.54mm grid so that the whole board can just plug into a giant protoboard if necessary
  4. more to come



GOOD PCB DESIGN GUIDELINES:

https://www.reddit.com/r/PrintedCircuitBoard/comments/1844gg9/4_layer_stm32f407_board_i_designed_its_for_high/?utm_source=share&utm_medium=web2x&context=3

  • Separate Analog and Ground planes
  • Use external precision reference voltages
  • Group analog and Digital domains together, don't mix them up
  • Routing should be neat. Do not ever use autorouter
    • "The shortest distance between any two points is a straight line" - Archimedes
  • Length match traces for all comms lines if possible.
    • For USB this is pretty much a requirement. Also for high-speed SPI
  • No routing under sensitive regions
    • Analog, RF, Crystals/clocks
  • Add plenty of decoupling caps and stitching vias
