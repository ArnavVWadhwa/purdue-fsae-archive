Purdue Formula SAE : ANSYS Structural FEA Basics

Created by Unknown User (reyno105), last modified on May 21, 2021

Video Example

Goes over FEA of simple part + bolted joint and lug analysis. Highlights good practices for meshing and joints

Geometry

  • The geometry brought in for FEA can have no sharp interior edges. These will create singularities that will not ever converge with mesh refinement. (The smaller your mesh gets, the higher the stress forever and ever. There's no asymtote.) 
  • Remove small external radii / chamfers. Sharp external edges will not create singularities. Removing your standard 0.010 inch chamfer / blend will have negligible effect on your results and will avoid issues with mesh sizing.
  • Think about things that will force smaller mesh sizes like thin walls, small radii, etc. If possible to remove these, analysis will become much easier.
  • If your shape is nearly constant thickness, try to make it fully constant thickness. This will allow the use of shell elements, which will cut down significantly on solve time.
  • Welds:
    • For an annealed metal, it is acceptable to directly model in welds. This is because there is no strength knockdown associated with heat effect. Do not use chamfers to model welds as the edges will be singularities. Instead, use fillets that are smaller than the expected weld. This is acceptable because of the ductility of metal. It is a reasonable assumption that any discontinuities will yield out. Analyze weld with correct tensile and yield strengths for given fill rod.
    • When analyzing welds on a cold worked or heat treated metal, the same process could be used if you assumed that the material becomes annealed. Otherwise, you have some work to do.

Supports

  • Generally go with the most accurate boundary condition that does not make your analysis non linear.
  • If possible use joints instead for reasons stated under that section.


Loads

  • If a load is being applied directly to a bolted connection, it is acceptable to apply it directly to the holes. The bolt and surrounding geometry is analyzed separately using hand calculation and empirical relations
    • If doing this, it is acceptable to not include the stress results for the elements that make up the hole. This is because they are over-rigidized to apply the load, so the stress will over predict.

Contacts

  • Depending on the type of contact, you will make you're model non-linear (non-linear geometry). If a stiffness change will occur as the model deflects, it is non-linear. Thus, a bonded contact does not cause a non-linear model while a frictional contact will. Sliding contact holds the surfaces together but allows them to slide. This does not cause non-linearity and is thus often a good compromise to create accurate load paths without increasing solve time significantly. 

Joints

  • As much as possible, use joints instead of supports. This is because connections allow you to see what nodes are connected to other nodes and give more obvious control over degrees of freedom that are fixed / free / have stiffnesses
  • Types of connections:
    • Can select various types, but all the info you need is shown by the degrees of freedom:
      • Select / create a connection
      • Select the type
      • Look at the degrees of freedom: a grayed out box means it's fixed while a color means it free (or has a stiffness)
        • X, Y, and Z are translation in those directions. RX, RY, and RZ are rotations about those directions
      • The bushing is particularly useful. It let's you directly specify stiffnesses:
        • Generally, stick to going down the diagonal axis of the matrix. This is each force/moment with respect translation/rotation in that direction. The other locations in the matrix would indicate coupling between directions (something like motion in the x direction creating force in the y direction).
          • The example above would be characteristic of a fastener that does not carry shear. The x-axis is aligned with the shank of the bolt. Thus, there is stiffness in the x-direction but non in the y or z-directions. There is not stiffness when twisting about the axis of the bolt (theta-x), but there is when bending the flange (theta-y and z). 
          • To calculate fastener stiffness, use paper called: "Influence of Fastener Flexibility on the Prediction of Load Transfer and Fatigue Life for Multiple-Row Joints"
            • Hopefully someone will make this calculator at some point for the team. In the mean time, using rigid connections is acceptable if you don't have bolts sharing load that have different stiffnesses. To do this, if bolts are close to each other and sharing load, make sure the bolt + flange stackup is the same.
  • Seeing connections (ha!):
    • Select solution information and then select the graphics tab. This will show which nodes are connected to which other nodes directly.

Meshing

Solid Meshing

  • For most simple parts, use a hex dominant mesh. This will lead to convergence with far fewer elements than a tetrahedral mesh.
    • Do this by applying a multizone mesh control to the body with a hexa mapped mesh type. Manually set the desired mesh size
  • For more complex parts, a tetrahedral mesh will likely be unavoidable due to the inability to create good hex elements.
    • Be sure to use quadratic elements. The long story short is that quadratic elements, though more computationally intensive per element, are much better at capturing stress gradients through the element. This allows far fewer elements to be used and will almost certainly come out net positive in terms of required solve time for convergence.
    • Turn on nonlinear mechanical for physics preference. This requires better element quality.
    • For final mesh, set jacobian ratio target of 0.66 (will take longer to mesh because it is going through and improving elements)
  • Best option for local refinement is often mesh sizing with sphere of influence

Shell Meshing

If you have a constant thickness, single surface part (think sheet metal), you're in luck. You can use a shell mesh. This can accurately capture the behavior of this shape with far few elements than would be required with a solid mesh. In NX, you can create midsurface faces (search midsurface to find the command). You can then bring these into Ansys and it will automatically know that a shell mesh is desired. This video covers it:

Mesh Quality Criteria

For where you are pulling stress results, aspect ratio should be less than 5 and jacobian ratio should greater than 0.8. For the most part, jacobian should stay above 0.66. There will usually be a few bad elements, the important things is to make sure there's not a bunch all together and that they're far from critical locations.

To see mesh quality:

  • Visually on part:
    • Select mesh and then select a mesh style:
    • Useful for confirming mesh is good in critical locations
  • As a histogram of all elements
    • Select mesh and then select mesh metric. A histogram will appear. If you
    • Useful for seeing if you model is generally refined enough to not cause issues
    • Also, you can select a bar in the histogram to show all the elements in that category. This is great for making sure you're bad elements aren't in important spots:

Solve Failure Troubleshooting

  • Run modal analysis first if you have much in the way of connections. If you have something that isn't properly attached or can't take load, modal will show it fly away / flop around. Static structural would just not solve.
    • If you have free body motion because connections are sufficient, you will have a mode at 0 hz and the loose part will be shown detached
  • If you get the failure "unkown failure, check output file..." - first thing to check is that you aren't requesting more cores than computer has.
  • If solve time is too high:
    • Usually the problem is having to large of a model to where the computer's ram is exceeded, forcing lots of shuffling. Check the solution output file for model size info
    • In general, we can run linear analysis with models that are larger than computer's RAM, non linear analysis with models that are below the computer's RAM, but NOT non linear models that are above RAM.


Convergence Study

The final test of element quality is a convergence study. Reduce element size until the stress results in the area of interest stabilize. Below is a plot of maximum stress vs. element size that indicates convergence has been achieved with an approximately 0.03 inch element size. A slow small rise in stress indicates convergence has been reached.

 (stress at critical location vs element size)


If adequate time is not available (so probably all the time), a reasonable rule of thumb is at least 5 elements to model a 90 degree radius and to go through the thickness of a given wall. Observing the averaged vs unaveraged stress contours is also an effective way to determine if mesh dependent results persist. Averaged results can often make bad meshes look good, while the unaveraged results tends to indicate issues. If possible though, run at least one analysis with elements size halved to confirm results.
