Purdue Formula SAE : i2 Standard / Pro - Using Telemetry (Testing Data)

Created by Unknown User (reyno105), last modified by Unknown User (dnocon) on Feb 15, 2022

  • Set up i2 Standard:

    • You'll need i2. Check out this page to get it: Downloading i2

    • To view data from our car, you'll want the template layout made by the suspension team (unless your subteam has their own they prefer). Get it here: MoTeC i2 Standard Layout Template

  • (New - 2022) Accessing Data

    • As of the 2022 Season, all of the vehicle / dyno runtime data is stored in a Microsoft Teams Channel. Team leads will need to grant you access as the channel is currently private. 
    • Documentation of vehicle testing days and corresponding data is documented here: Test Data Notes
  • (OLD - Pre-2022) Accessing Data
    • All our data is stored here by the electronics team: https://drive.google.com/drive/folders/1PXzK-UU0GXKrBrn_7jVycIxSJRm_IwWE
    • During our most recent testing, we added better documentation of what we tested and what data files correspond to what tests. This is documented here: Test Data Notes
    • Example: 
      • A good sampling of data can be found here. Download the entire folder circled.
      • It will download as a .zip, so extract it
      • Open data by going to file → open in i2 standard
        • These two files have good data to look through. They are the car with and without the aero package going around the grand-prix track:
  • To learn:
    • The help information provided in the program is pretty good. There are also many tutorials on youtube, but I can't speak to how good any of them are.
    • Try to know how to open data, navigate the data (zoom + scroll), change what is plotted and where, apply filters and factors to the data, and overlay data from multiple runs.
  • i2 Standard has many good features, but it can't take derivatives, do integrals, do more advanced filtering, etc. If you want to export the data to excel and do any kind of fancier formatting, take a look at this postprocessing script confluence page  from suspension to get started: MoTeC Data Plotting Tool in Matlab (Shockpots)



MATLAB - Using Telemetry


Although i2 is useful for preliminary viewing of test data, for a more in-depth analysis of the data, we will turn to MATDAB. This means you'll have to export the data form i2. Do this by highlighting the segment of data of interest, right click and select export data. Be sure to select "All Channels" and choose the csv format. The analysis script that we use is on the formula network drive, a (likely outdated, but still usable) version can be found here:

Track_Dev.zip


To start, in MATLAB, open the csv file of interest. It might take a while depending on the length of data you exported. A window should open looking like this.
