Purdue Formula SAE : Explanation of Matlab code and areas of improvement

Created by Unknown User (mkotur), last modified on Apr 19, 2020

MATLAB codes have been written that do these calculations for you, but it is important to understand these equations, where they come from, and why we use them so that you can correctly change the code to calculate values for new layup schedules.

 

Here are the most important parts of the MATLAB code that you should change depending on your layup schedule and the different carbon you use.

To begin with you need to select how many different types of carbon you have material properties for and that you are using for your layup, Figure 1 shows where to input the number of different carbon materials:

Figure 1. Setting the number of different carbons used

The next step is to setup the material properties for each of the carbons you are using as follows. Make sure that all the units correspond to what is commented out on the top of Figure 2. Furthermore, do not forget to increment the number given to the ply_properties by 1 for each additional carbon used. You want to do the same thing with the core material you are using, even though the code only uses the thickness given to the core for now (future members may be able to improve the code to add all the properties of the core).

Figure 2. Adding carbon and core material properties

Finally, you need to set your layup schedule up by adding the number of plies in each schedule, the orientation of each ply, and the carbon used for each ply as follows:

Figure 3. Layup Schedule

In Figure 3, the theta vector represents the orientation of each ply and the ply_order vector represents the number of plies (in this case there are 4) and the carbon you want to use (from Figure 2, the “2” represents the unidirectional carbon). Once you have finished setting up your layup schedule you can run the script and it will output the important material properties discussed above. (NOTE: we were unable to find a reliable method to find the ply failure force and we ended up not using MATLAB to find the failure force. However, future members may try to find a way to integrate the failure force in the code).

Figure 4. Output of MATLAB code

Figure 4 shows the outputted results of the MATLAB code for the unidirectional layup used as an example above ([0UD 45UD 90UD -45UD]) for a panel with a half inch core.
