Purdue Formula SAE : Scholar Sub Cluster B

Created by Unknown User (rkumaran) on Aug 23, 2021

There's a new mystery sub cluster of scholar nodes with 64 cores each that seem to have magically popped up at some point during the summer of 2021. 

I cannot find any documentation for this new sub cluster B, and it does not seem like this sub cluster has been fully activated. 

Up until a few minutes ago, fsaeSTAR assumed that every scholar job would feature nodes that all had 20 cores each. This assumption does not hold when you could potentially have 64 core nodes thrown at you. 

From now on, fsaeSTAR will apply a constraint to only use sub cluster A nodes (the ones with 20 cores), will no longer support running multiple sims on the same job (too confusing to split resources), and will pull the number of available CPUs from SLURM and pass that number on to STAR, rather than use the user inputted CPU count.
