Purdue Formula SAE : KPI Summary Dev Notes

Created by MacNab, Zachary Robert, last modified on Mar 24, 2023

The purpose of this page is to provide a notebook for anyone wanting to use the KPI summary tool in the future. Just a place for devs to write down any though processes and such.

Tool Purpose

The purpose of this tool is to better understand and process recorded car data. Currently, I think that MoTeC isKPI stands for Key Performance Indicator. 

Requirements

  • Needs to be robust to accommodate any CSV from MoTeC i2. 
  • The user shall NEVER have to interact with the code for normal operation. 
  • There should be "templates" for quick data analysis at the track. I.E., there should be a "skidpad" report, an "acceleration" report, etc.
  • laps should be automatically parsed, and the user should be able to select what laps they would like to see. 

Wants

  • Math channels should be able to be created from the GUI interface
  • Be able to create templates within the GUI 

Thoughts

Python will be used for this project to generate PDF reports. Workflow should be pretty simple:

  1. Export Data as CSV from MoTeC
  2. Import data to KPI Program
  3. Pick your template and your laps that you want to view
  4. Hit generate and it will generate the report
