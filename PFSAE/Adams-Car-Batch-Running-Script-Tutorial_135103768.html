Purdue Formula SAE : Adams Car Batch Running Script Tutorial

Created by Unknown User (reyno105), last modified on Dec 01, 2019

Overview:

This script is used to generate the GG diagram that gets passed into the quasi-steady-state lap time simulation that lives in Matlab. It was originally designed to create entire GG diagram outright that has pure longitudinal, pure lateral, and combined lateral longitudinal accelerations at each speed. However, in the current simulation workflow, it is only used to provide pure lateral and longitudinal accelerations at each speed. Then an ellipse shape is assumed to determine the combined acceleration envelope. This is one of the major assumptions / weaknesses of the lap time sim in its current form.

Adams Car Quasi-Steady State Simulations:

A quasi-steady sate simulation is looking for a steady state that the car can exist with certain parameters and targets. For example, a simulation may attempt to achieve a lateral acceleration of 1.7g at a speed of 45 mph. The speed will be set at 45 mph, and the lateral acceleration will be increased from 0 to 1.7, assuming that 1.7 is possible for the vehicle. If it is not, the simulation will terminate after a set number of iterations at a lower acceleration. This may be used to find the theoretical maximum acceleration of the vehicle, and is generally how I have used these simulations. There are two types of simulations required for the lap time simulation: lateral acceleration at a constant velocity and longitudinal acceleration a constant velocity. The longitudinal acceleration will include both an acceleration and a braking simulation.


Running Adams Car Non-interactively:

This script sends commands to the windows command prompt to run Adams Car separately from the normal GUI. So it is necessary to understand the Adams Car Simulation file structure and naming as well as how those commands work.

For every Adams Car simulation, there are two files required: a .acf and a .adm. The .adm has all the vehicle parameters. It points to a bunch of files that contain the information for all the various components of the vehicle such as a torque curve file, tire model files, shock and damper curve files, a chassis file with weight and cg, etc. The .acf file contains the commands in the Adams language to run the simulation. It points to the .adm file, the working directory, and the license file and then has the commands required to run the desired simulation.

So, for simulations of the same vehicle, the .adm file can be the same. Only the file name will change.

The .acf file, however, will need to be changed for each simulation to call out the desired speed, target acceleration, etc.


The Code Itself

This script is in Python 3.X. I recommend using spyder. Not only is this a pretty decent ide, it also comes with all of the modules needed to run this script (which is nothing too fancy, just matplotlib, numpy, pandas, etc.)
