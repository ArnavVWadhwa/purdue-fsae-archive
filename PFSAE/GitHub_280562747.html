Purdue Formula SAE : GitHub

Created by Kurisko, Keith Gregory, last modified on Dec 07, 2023

GitHub is one of the most popular resources for developers to share code and work on projects together. It’s free, easy to use, and has become fundamental in the movement toward open-source software. As our FSAE team moves towards more collaboration software development and modeling it is critical to have a foundation to manage all of this work. GitHub is our answer to this. 



How to get added to the Team GitHub:

  1. Go to GitHub and create an account if you do not already have one
  2. Reach out to your Team Lead to request access
  3. Team Lead can go to Purdue-FSAE (github.com) and Invite Someone
  4. It is recommended to download the desktop version of GitHub to use commands and interface with GIT Repositories



How does GitHub Work?: 

All the files are stored on a GitHub server that any developer can create a development branch from the develop branch and work on sections of code or the model independently. Once finished and proved out the changes can be reviewed and pushed back to the main server where the changes are merged to the develop branch. Once ready for release the master branch can be updated from the develop branch. Use the master branch to run all models as this is the "Source of Truth". Note that merge issues do happen and they can be reversed which is the great thing about Git. Also any change can be reverted to if the model breaks or something was not supposed to be pushed to the master branch. GitHub has plenty of documentation on how to do this. If you want you can run Git commands from a command line to streamline the process but this is not necessary. It is a good thing to learn as it is standard in industry for software development. Follow the instructions here if you would like GitHub CLI quickstart - GitHub Docs.
