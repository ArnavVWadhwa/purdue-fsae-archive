Step 1:

Read Tune to Win by Carroll Smith. The suspension team is tasked with making the vehicle architecture decisions and compromises that affect all other systems. Accordingly, and all around understanding of the car is a must. Tune to Win is the best start at that understanding. It also places emphasis on suspension analysis that will help start building the intuition required to make it through the math that comes in Race Car Vehicle Dynamics. The good news is Tune to Win is a short read. You can buy a hard copy for ~$20, or read the PDF in the above google drive folder.

    • If you want to dive deep:

Structures Analysis

  • If you have not taken Mechanics of Material (ME 323 or aero / civil equivalent):
    • Focus on vehicle dynamics until you take this class. Your team lead should be able to help you do the structures analysis you need to until then. If there is an ME course worth taking asap, it is almost certainly ME 323. It's the only class that I have had brought up specifically in interviews.
  • If you have taken Mechanics of Materials: 
    • Bolts:
      • threaded fasteners for FSAE: https://www.fsaeonline.com/cdsweb/gen/DocumentResources.aspx
      • bolted joint section of shigley's machine design: google search for pdf
      • NASA-STD-5020: just google search. NASA makes there manuals public
    • Lugs:
      • lug section of airframe stress analysis and sizing: online version available through Purdue library

  • NX 12:
    • About:
      • This is the computer-aided design (CAD) package the team uses for all design
      • It is one of the go to CAD packages for the automotive and aerospace industries. Daimler, FCA, General Motors, Boeing, and SpaceX all use it.
    • To Access
      • It can be found on the ME computers, as well as any ITAP computer. The team also has some licenses for it to be installed on personal computers. Ask your team lead about this if required.
    • To learn NX basics: 
      • Make sketches, then solids bodies, then assemblies. Drawings for these are provided in the CAD practice 2 pdf in the google drive folder
      • Tutorials are readily found on youtube.
      • To assess your readiness to design parts for the team, make the components and assembly in the vblock2012_upload pdf in the google drive folder
    • To learn the team's NX practices:
      • FIXME hopefully something on the confluence
  • ANSYS Workbench:
    • About:
      • This is the finite element analysis (FEA) software uses in most cases.
      • It is used by almost any company that does FEA. Most companies would also have other FEA packages. ANSYS is great for analyzing small individual components or assemblies where problem size is not an issue. For large systems, most companies would turn to a different package. It was the primary FEA package at Hendrickson; it was one of several used at SpaceX.
      • Any FEA setup includes a pre processor, solver, and post processor. ANSYS Workbench includes all three. Other workflows / programs may have these separated out.
    • To access:
      • The student version is free on ANSYS' website: https://www.ansys.com/academic/free-student-products
      • Working on getting team licenses now, but may have a limited number - FIXME
    • To learn:
      • This series of tutorials is useful: https://studentcommunity.ansys.com/thread/fea-webinars-for-fsae-teams-hosted-by-leap-australia/
        • Use RL_UPRIGHT_V1p2 step file for geometry to practice on.
      • The brake pedal video here is also good: https://studentcommunity.ansys.com/thread/fsae-baja-video-tutorials/
  • Adams Car:
    • About:
      • This is what use for suspension kinematics, vehicle dynamics, and suspension loads
      • It is used by just about every single automotive company. The more general version (for analyzing any rigid body motion) is also used by aerospace companies:
    • To access:
      • It is available on computers in the 2nd floor ME lab
      • These computers can be accessed via remote desktop: https://engineering.purdue.edu/ECN/Support/KB/Docs/StudentPCsavailablefo
    • To learn:
      • This video gives a great introduction:
      • The FSAE template can be found here: https://simcompanion.mscsoftware.com/infocenter/index?page=content&id=KB8020723
        • The team's model is built off of this template, and will generally act the same.
  • Matlab: 
    • About:
      • This is an ME favorite for programming. The programs for post processing FEA results to create load time histories at SpaceX were in Matlab. However, many companies try to limit its use because it costs big money and can usually be replaced by an open-source language like python
      • It is most convenient when dealing with matrices or doing a lot of plotting. However, python can replace essentially anything it does through modules like matplotlib, numpy, and pandas.
      • The heart of the team's lap time simulation is written in Matlab.
    • To Access:
      • This is on any ME or ITAP computer.
      • Can also get a license on your PC thru Purdue: Matlab
    • To learn:
      • This gets introduced by Purdue FYE
      • The documentation on the mathworks website is really good
  • Python
    • About:
      • This is probably the most common programming language among MEs. The programs used at SpaceX for doing fatigue analysis and bolted joint analysis for large structures were all in python, as well as many other programs.
      • On the team side, the code used for batch running Adams Car simulations for the lap time simulation is written in python. Many data post-processing tools that will hopefully exist some day would also likely be python.
      • Is often a great way to create value as an intern, because many engineers who are not familiar with coding can be significantly benefitted by simple scripts for data collecting, sorting, filtering, etc. 
    • To access:
      • Download the python 3.7 anaconda distribution here: https://www.anaconda.com/distribution/
        • The team uses python 3
      • Use the ide called spyder (there are a bunch of other options, but this one is a favorite for various reasons)
    • To learn:
      • There are tons of good tutorials, but this one looks nice: https://www.learnpython.org/
        • The data science tutorial go over two of the best modules for making python do way more big brain stuff
      • You could also look at the recommendations here: https://wiki.python.org/moin/BeginnersGuide/NonProgrammers
      • Python's documentation is great: https://docs.python.org/3.8/library/index.html
      • Good modules to look into are:
        • Numpy
        • Pandas
        • Matplotlib
  • MoTeC i2 Standard
    • About:
      • MoTeC is one of the biggest names in motorsports electronics. They provide the electronics packages that get used from amateur up to professional racing including Rally Cross cars, IMSA comptetitors, etc. I2 is MoTeC's software for analyzing vehicle telemetry.
      • We use this software to analyze the telemetry from our vehicles. This is used for everything from correlating the lap time simulation to testing suspension loads.
    • To access:
      • i2 Standard is free from MoTeC. Download here: https://www.motec.com.au/i2/i2downloads/ (note, do not download pro as it requires licenses)
      • To view data from our car, you'll want the template layout made by the suspension team. Get it here: MoTeC i2 Standard Layout Template
      • You'll also want some data to dig through. all our data is stored here by the electronics team: https://drive.google.com/drive/folders/1PXzK-UU0GXKrBrn_7jVycIxSJRm_IwWE
        • Good sample data can be found here. Download the entire folder circled.
        • These two files have good data to look through. They are the car with and without the aero package going around the grand-prix track:
    • To learn:
      • The help information provided in the program is pretty good. There are also many tutorials on youtube, but I can't speak to how good any of them are.
      • Try to know how to open data, navigate the data (zoom + scroll), change what is plotted and where, apply filters and factors to the data, and overlay data from multiple runs.

Machining:

Starting about half way through fall semester, most suspension work is in the machine shop. You can get access if you are in Mechanical Engineering OR have been granted access through the team. To get access, you need to take the building access quiz (https://engineering.purdue.edu/MECL/building/) and the machine shop access quiz (https://engineering.purdue.edu/MECL/projectshop/).

The machine shop hours are listed here: https://engineering.purdue.edu/MECL/projectshop/
