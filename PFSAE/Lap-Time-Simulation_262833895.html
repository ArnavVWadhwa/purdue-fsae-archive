Purdue Formula SAE : Lap Time Simulation

Created by Kurisko, Keith Gregory, last modified on Dec 06, 2023

Concept Overview:

A Lap Time Simulation tool allows for the study of various vehicle concepts and designs. This is possible by running multiple vehicle configurations through the same tracks or laps and analyzing the differences between lap time and velocity traces. In Formula SAE, there are clearly defined rules and equations that turn lap times from each dynamic event into the points equivalent which will allow use to quantify how many competition points we could theoretically gain by making a specific architecture decision. The Lap Sim can be used to correlate track data to vehicle models especially with the Grand Prix track as this is where we can compare tests to the same study in a simulated environment. This is a good way to validate designs along with our models.

A well explained and documented page on Lap Simulation can be found: Lap Time Simulation | Jonathan Vogel (jovogel.com)

Model Limitations:

As with all simulations and models, no model is accurate but some models are useful. It is important to understand the limitations of your model and assumptions being made in order to gain useful information. 

The Lap Sim is a 1:1 comparison of a certain vehicle configuration to another. As long as the theory used behind the vehicle model is the same then results can be used to make informed decisions on vehicle architecture and designs that are faster and will gain more points. It is important to recognize that the model we use cannot look at kinematic changes as our tire model is limited by extrapolation specifically with camber data. This will lead to inaccurate results that also wont make sense. Sensitivity studies should be conducted to evaluate the best vehicle architecture. 

Inputs:

  • Tracks
  • Vehicle Parameters to feed into GGV
  • GGV model to run simulation

Outputs:

  • Lap Times
  • Comp Points
  • Telemetry traces
    • Speed
    • Accelerations
    • Downforce
    • RPM
    • Torque/Power
    • Gear Position
    • Fuel Usage
    • Etc.
  • Cycle Count for Fatigue

  1. Configure Vehicle Parameters File
  2. Run GGV

  3. Input GGV and Track (if needs to be changed) and run Lap Sim
  4. Take Print Out Points Results or Study Plots that can be made referencing telemetry variables
