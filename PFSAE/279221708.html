Purdue Formula SAE : Design Review Guidelines | ELE

Created by Paradise, Hunter on Nov 14, 2023

DCR:

  • All team design reviews


Software

  • Declared backend/software choice


Hardware & Prototyping (Same as hardware but follows typical review order scaled to project schedule)


PDR:

Software (Delayed relative to all team)

  • Flowcharts/Pseudocode
    • Priority levels for features
  • Rough user interface plan
  • Space efficiency implementation
  • Workflow determined for multiple people


Hardware

  • Schematics
    • Physical: Layouts
    • Hardware: Pinouts
    • Prototyping: PCB/Wiring
  • Rough cad (if applicable)
  • Components placed
  • Space allocation/sizing
  • Future-proofing


CDR:

Software:

  • Working product in hand (not fully debugged)


Hardware

  • Finalized CAD and layouts (Ready to be manufactured)
  • Firmware flowcharts/pseudocode


MRR:

Software:

  • N/A


Hardware

  • Final design overview
  • Finalized manufacturing/ordering timeline
  • Manufacturing overview


TRR (Less about validation more about debugging):

Software (Shortly after CDR):

  • Full debugging plan
  • Test scripts
  • Testing timeline


Hardware

  • Full debugging plan
  • Test cases
