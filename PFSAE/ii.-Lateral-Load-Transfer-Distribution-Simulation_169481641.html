Page Description

This page discusses the only technical changes for the torsion test model to gain an LLTD tool

Modeling Springs in Mechanical

Instead of using the rigid square beams that were valid for the torsion test model, we'll use spring connections. In order to do this, we need a remote point as shown below to reference relative to the frame body.

Create inboard spring remote points based on the coordinates within NX. Attach these points relative to the nearest vertex to the intended tab location.

Modeling Tires in Mechanical

Using spherical Body-Ground connections was found to yield the most realistic results for total lateral load transfer. There is likely a better way to solve this connection.

Set the spherical connection type to Body-Ground.

Change the Define By section from Geometric Selection to Global Coordinates. Set the Origin Z to ground level.

Adding Representative Masses

Add point masses to represent the driver, engine, and differential. Additionally, it may be useful in the future to add masses at the wheel centers to represent the unsprung mass (e.g. tires, wheels, uprights). At this point, we've accounted for the mass of the driver, engine, differential, and suspension assembly. We need to account for the rest of the mass, which may be distributed evenly across the car. This is done by changing the density of the tubes so the total weight (including point masses) equals the weight of the car and driver.

Static Structural Inputs / Outputs

Apply the standard earth gravity, and apply a 1.65g acceleration in the Y direction to simulate a corner. For outputs, probe the spherical Body-Ground "tire" connections for the final Z load. Make this an output parameter in order to be able to export it to .csv later.
