Purdue Formula SAE : Running Full Vehicle and Suspension Simulations

Created by Klaveano, Alexander Ryan on Dec 04, 2021

This page will give a brief overview of common simulations in ADAMS Car and how to configure and run them.


Before running any simulations, make sure to follow the instructions on Page: "Adams File Setup" to configure the software with all of the right file paths to the vehicle files. Select an appropriate directory for where you want your simulation files to go. (Each simulation will produce six different files, so this tends to clutter up directories, it's generally a good idea to make a new folder in whichever location you're working, in which all of your simulation files will go.

Assembly Selection

The first step is obviously to open your assembly to be tested. Right clicking on the "Assembly Name" field in the Open Assembly dialog box from File → Open → Assembly allows you to browse for an assembly file. Full Vehicle Assemblies located in Z:\Knowledge_Base\01-Suspension\06-Adams_Car_Vehicle_Files\private2


Make sure your units are proper for your application. If getting sims for the Lap Sim, use MMKS, otherwise, IPS is the way to go. These can be changed from Settings → Units

File Name Convention

To keep simulation output files organized, we use a file naming convention to differentiate outputs from different simulation runs. When running a simulation in ADAMS, the dialog box for configuring simulation parameters has a field "Output Prefix". In this field you should enter the following [Assembly or Subsystem Name]_[Simulation Type (this can be an acronym)]_[Simulation number]. For example, the first run of a Static Vehicle Characteristics (svc) simulation on the assembly "PF21B_13_wa_r", would have the output prefix "PF21B_13_wa_r_svc_1" as shown below. This isn't too critical to keep consistent, but it's good to be able to differentiate between sims that are in the same directory, and this is the easiest way to do that.
