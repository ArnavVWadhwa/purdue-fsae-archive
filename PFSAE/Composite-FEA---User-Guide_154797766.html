Purdue Formula SAE : Composite FEA - User Guide

Created by Unknown User (green338), last modified on Dec 10, 2021

https://www.mm.bme.hu/~gyebro/files/ans_help_v182/ans_cmd/Hlp_C_SFE.html 

This page is intended to cover the specifics of modeling composites using FEA. Any specifics of overall FEA best practices are on this page: ANSYS Structural FEA Basics.

Currently we use ACP (Ansys Composite Pre/Post) for our analysis. This is primarily due to availability of the software and familiarity. This software also allows for somewhat "simple" integration of pressure loads from our CFD into our FEA models.

Overall:

You should always be using shell meshes with composite parts. All of our carbon components are near constant thickness, single surface parts which makes shell meshes ideal from a solution time standpoint. Shell meshes are also required by our CFD pressure export and ACP. Unlike with sheet metal parts, the outer surface of the composite part (not the midsection) should be used. This is due to the fact that ACP adds thickness along a chosen normal to the surface, similar to how actual composites are manufactured. CFD pressure exports also require a surface. As pressure acts on an area, you must give STAR a pressure surface to export rather than a volume.


Simulation Process (example part - rear wing main element): 

Obtaining geometry

The FEA process currently in use requires 3 pieces of software. These are Siemens NX (CAD and meshing), Star-CCM+ (CFD), Ansys Workbench (FEA).

The geometry that's being analyzed should be a surface only. The surface that is controlled by the tooling (mold surface/A surface) should be used as the analyzed surface. In this example, the outer surface (exposed to airflow) of the rear wing will be used.




Above left: rear wing main element surface being used in orange, rear view. This surface is being wave linked to a separate .prt file to avoid attaching a .sim file to the car geometry.

Above right: isolated rear wing main element geometry


All surfaces that are being used should be exported. If multiple carbon components are being analyzed (flaps attached to endplates, bonded sections) all of these components should be exported in the same .prt file.


Meshing/Exporting

We use NX to mesh our parts as it can be exported as a .dat file that can be read by both Ansys and Star-CCM. Since NX doesn't allow for easy parametrization of the mesh, dependency studies should be completed in Ansys first with representative "simple" loads (forces and pressures) to determine the minimum mesh size that achieves independence. For this example, the mesh size has already been determined, so the dependency study will not be covered.

It's usually easier to open the surface that is being analyzed in Simcenter 3D. Switching from Modeling to Pre/Post in NX will hide all other parts and won't allow for easy adjustments to the geometry. Having NX open with geometry and SC3D open with the mesh is the preferred method.

Open the analysis .prt in Simcenter 3D or NX. Go to the Application tab in the ribbon and change from Modeling to Pre/Post

Next, click the "New FEM and Simulation" button.

All options can be left as the defaults. In the Home tab on the ribbon, select "2D Mesh." Select the surfaces that need to be meshed and enter the desired base size, as determined by your previous mesh dependency study

The part will mesh automatically. If it doesn't, the "Update" button can also be used to generate the mesh.  The "Mesh Control" menu can be used to add face, edge, and point sizing. For this example, these features will not be used, since the rear wing is relatively simple geometry. Below is an image of the meshed rear wing.

Once the surface has been meshed, the file is ready to be exported. Go to File > Export > Export Simulation. Name the file and save to your chosen directory. The resulting .dat file will be used for both ansys and star-ccm.

This part will also use a set of ribs to act as boundary conditions for the FEA. These are exported as parasolid files and can be meshed directly in ansys.

Mapping Pressure Loads

Open the .sim file that includes the geometry you would like to analyze and is the correct operating conditions for the analysis. Geometry must match exactly, otherwise the mapped pressures won't correlate to the aero loads being placed on the part. Once the sim is loaded, select File > Import > Import CAE Model. Choose the .dat file exported from NX and select inches as the import unit if not selected already. It can be helpful to open a geometry scene to verify that the imported file is in the correct location and is the desired geometry. Below is the rear wing example part in star-ccm.

Right click on the imported model (labeled in the above screenshot as "Nastran: rw_455_video_nx_export") and select "Map Data > Surface Data." In the dialogue box that comes up, select the positive imported surface as the "Imported Surface," and the "Subtract" region as the boundary. This will map pressure to each cell of the mesh. Click apply and wait for the values to be mapped - this should take no longer than 2-5 minutes for a reasonable sized mesh on single elements, but may take longer for larger components such as the undertray or full wing assemblies. 

Once the loads have been mapped, they are ready for export. Right click on the imported model again and select "Export Mapped Data..." Select the positive face in the first tab, and then choose "Pressure" and "MappedPressure" with psi as the units on the second tab. The export options should match what is shown in the below images.

The file will be saved as a .nas file. In file explorer, change the extension to a .csv and open in excel. Create a new tab and copy over the cell number (column C) and load (column D) to a new sheet. All other values in the sheet can be manually entered. The format should be as follows.

SFE,Cell number,1,PRES,Load,Load,Load,Load

An example of the format is shown below.

Save the file as a .csv file again under a new name. This will be used to copy/paste into ansys as APDL commands.

Defining the Layup

In ansys, add the "External Model" block to the workbench. This will be used to import that .dat file with the mesh from NX. Go into setup and select the location of the .dat file exported from NX and change the units to "Consistent BIN," and then hit update. This will import the model. Add the "ACP Pre" block to the workbench, and connect the model tab of the External Model to the model tab of ACP Pre. This will transfer the mesh to the ACP block. Finally, add your materials in the Engineering Data tab of the ACP Pre model. For our carbon, we are typically using materials that match "Epoxy Carbon Woven (230 Gpa) Prepreg" or "Epoxy Carbon Uni (230 Gpa) Prepreg" for both infusion and prepreg. Our resin content when using infusion matches that of prepreg, and the material properties should match as well. If your part is using honeycomb, it should be aded as well, but the example component will not use it. At this point, the ansys workbench should look like the following.

Open the model section of the ACP (Pre) block. This will bring up the Mechanical window. First, a thickness must be added to the mesh by clicking on Geometry > (imported surfaces. this may be more than one, depending on what parts are being analyzed). In the bottom left hand corner of the window, add the "Thickness" parameter This thickness will be replaced after the layup is defined, so any value will work. In the same window, change the offset type from "Middle" to "Top" and verify that the offset direction is correct for the layup direction. This will determine which side of the surface is thickened after the layup is defined. For some geometries, it may need to be changed to "Bottom," however for the example part it will be top.

Material assignment doesn't matter for this part either, as it will be replaced. If the layup will be split by face (different layup schedules for different areas), named selections should be used. Click on all faces that will form a layup area, and then right click in empty space and select "Create Named Selection." This will be passed on to ACP and allow for the faces to be selected as a group and assigned a different layup schedule if desired. Close mechanical (and save), and then update the project. This will pass the mesh on to the ACP preprocessor. Once the project has updated, open the ACP Setup tab. The following interface should appear.

Open the "Material Data" dropdown and verify that all materials selected in the engineering data tab were passed through in "Materials." Right click on "fabrics" and select "Create Fabric" to bring up the fabric creation dialogue. Select the material that the fabric will be and the thickness. For composites, the Hexcel Cured Ply Thickness Calculator can be used to get an estimate of laminate thickness (https://www.hexcel.com:82/calculators/src/CPTCalculator.aspx). For all of these, void content should be set to 0, as we have no reliable way of measuring this quantity. The lack of data on void content should instead be used as a reason to include additional margin when determining acceptable strain limits on composite structures. Include all fabrics that will be used in the layup in the fabrics dropdown. For materials such as core, where a thickness is given from the manufacturer, the thickness should be set to the true value (ex 4mm core should have a thickness of 0.004m). Expand the "Stackups" dropdown and click "Create Stackup." Add plies to the stackup in the desired order and orientation. All other options can be left as the defaults. Each layup zone on the part should have its own stackup, to allow for easy modification of the layup schedule if changes need to be made. For the example part, there is only one zone, so only one stackup will be made. Below is the tree after all stackups have been made.

Rosettes determine the direction of the carbon relative to the part geometry. For the rear wing, the 0 degree direction is intended to be along the chord of the wing, and the 90 degree direction should be along the span. Right click on "Rosettes" and  select "Create Rosette." Select an origin on the part as a reference, and adjust the principal axes if needed. An example dialogue is shown below.

A rosette should be created for each layup zone on the component, with the origin in the element set being used. Once this has been completed, right click on "Oriented Selection Sets" and select "Create Oriented Selection Set." As with all other steps in ACP, an oriented selection set must be created for each layup zone. The element set that corresponds to each zone should be used for the oriented selection set. Choose the corresponding rosette and element set, and choose a point on the component as a reference for the layup direction under "Orientation." This typically needs to be flipped, with the arrow pointing in the direction of material buildup. Below is a photo of the global oriented selection set used for the rear wing.

Finally, the layup schedule can be applied to the part. Right click on "Modeling Groups" and select "Create Modeling Group." Expand the newly created modeling group (only one required per analysis run), and create a modeling ply for each layup zone. Select the oriented selection set and layup schedule corresponding to each zone. This will assign plies to each area. Below is the selection used for the global rear wing main element modeling ply.

Click update on the toolbar above the geometry window to assign thicknesses to the part geometry. Verify that the desired thicknesses are present in each zone of the part. For this component, a global thickness of 0.00128m was desired.

Save and close the ACP window.

Solving

Add a "Static Structural" block to the workbench. Drag the ACP "Setup" tab over to the Static Structural "Model" tab, and click "Transfer Shell Composite Data" in the dialogue menu. At this time, any solid modeled copmonents (ribs, braces, tabs) can also be added to the static structural block. These components should be meshed in "Model" blocks separately from the structural tab. General solid modeling FEA best practices should be used for these, which are covered in the global team ANSYS resources. At this point, the workbench will most likely look similar to the following.

Update the model to transfer all mesh and layup information to the static structural block. Open the model tab in static structural. The meshed composite part should be present, with an "Imported Plies" dropdown on the model tree. Below is a view of the rear wing example part.

The CFD loads will now be imported. Add a "Commands" input in the Static Structural. One commands input should be added for every 80,000 elements in the model, due to a limit on the number of lines that can be input at once. The .csv file with the loads that was generated earlier should be opened in notepad. The first 80,000 lines (less can be selected, but more than 80,000 will cause issues) should be highlighted and copied. Paste them into the ansys command input that was added. The first 80,000 lines of the example part are below.

Repeat for each 80,000 lines until all values in the notepad have been entered into ansys. Lines cannot be copied, as this will apply the pressure twice to the same node. The rest of the boundary conditions can now be added, such as fixed supports, external loads that are not captured by CFD (such as inertial loads, loads from other aero parts not in the current model, or supports). The values to solve for can also be added. Deflection acts in the same manner as with conventional FEA. For stress and strain, results can be given by layer or ply. The layer option will provide results for the entire laminate (most useful for strain, as it is consistent throughout the laminate thickness), while ply can be used to select a specific ply to obtain results for (useful for stress, as this varies by ply). The model can now be solved. Views of the example part deflection and strain outputs are below.
