Purdue Formula SAE : Inventory Management Planning | PF24

Created by Paradise, Hunter, last modified by Aadi Biyani on Nov 02, 2023

Code Architecture:

  • App
    • Apple not the same as Android
    • XCode
  • Website
    • Python 
    • Usable on any device with internet
  • Computer Application
    • Limited to windows → Inaccessible
    • Any language


User Interface:

  • Potential Options for app: React and Flutter
  • Users should be able to "login" (some credentials needed for checkout and checkin)
  • Users should be able to update inventory list
  • Budget Interface
    • keep track of all inventory we could buy
    • money tracker and optimization of costs 

Tracking System:

  • Track amount of inventory used user input of percentage left
  • track where item is?
    • kind of hard would need something on each inventory item
    • could use google maps API with python if python is used

Database Structure:

  • VERY IMPORTANT 
  • could be done in app. Once app is designed each item could be added through updating the inventory list.
    • would be easier to update year to year (would not have to create new database each year)

Search Function:

  • Search should work like regular searches but there should be some filters that need to be added
    • filters: type of inventory, how much of said inventory is needed, etc. 

Check In/Check Out Feature:

  • Should work like a library check in and check out system. 
    • Users MUST update inventory after checked back in (amount left, any changes, etc.)

New Item vs. Check In:

  • Two sections update inventory and check out inventory
    • update only some users can do and must be approved by team leads

Stock vs. Components:

  • I think all inventory items should be split into categories and the users will have specific functions for each function.

Hardware Management:

  • ALL INVENTORY MUST BE MANAGED 
  • the app would not be useful or provide any information if hardware is not managed

Ease of Life Structure:

  • Must have easy and crisp functionality and should not crash or have any problems.
