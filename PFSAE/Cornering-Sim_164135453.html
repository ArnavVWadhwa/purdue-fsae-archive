Purdue Formula SAE : Cornering Sim

Created by Unknown User (agarw162), last modified by Zhang, Chenkai on May 07, 2023

This page talks about the differences between the cornering environment, and the straight-line environment. The cornering environment was developed with the intention of keeping it as similar as the straight-line environment as possible, so hopefully this will be a pretty short document.  

An immediate change is the cornering environment no longer features a block part for the domain, but rather a 3-D CAD parameterized Freestream_C block. The “_C” postfix is incredibly important, as fsaeSTAR looks for this postfix to decide whether or not to give the simulation a straight-line or a cornering simulation treatment. The 3-D CAD model is parameterized, so modifying cornering geometry like cornering radius, or domain sizing can be done through STAR-CCM+ parameters, rather than requiring fsaeSTAR to do anything whack in the 3-D CAD environment, or requiring the user to have to manually modify cornering geometry.  

The other main change with the environment is how flow velocities are handled. With a cornering sim, the a rotating reference frame is required. Crosswind is disabled for cornering sims to avoid complications with the reference frame. All domain boundaries are set to the cornering reference frame, with inlet velocity 0 m/s.
