High level tool workflow

CAN Tracking Update

CAN tracking sheet is updated in confluence

→ Manually update CAN tracking excel sheet

CAN Snapshot Update

CAN Snapshot taken with PCAN-View

→ Export snapshot data

→ Import snapshot data into excel and save as a .csv

CAN Debugging Tool

Pull most recently updated CAN tracking excel file

→ Store each column in arrays

→ Ask user for CAN snapshot file path

→ Open CAN snapshot file and store each column into arrays

→ Compare arrays containing same data (Compare CAN ID arrays, compare ID lengths, compare ID frequency (within an error))

  1. Take first ID in snapshot file and search CAN tracking excel sheet for matching ID
    1. If there is a match:
      1. Check for ID length and ID frequency match
      2. If these don't match, store ID in an array containing "error" IDs and store error type (ID / Length / Frequency) in associating array
    2. else:
      1. store ID in an array containing "error" IDs and store error type (ID / Length / Frequency) in associating array
  2. Repeat step 1 until all IDs in the snapshot file are checked

→ Output a report to the user containing every error ID and the error type associated with that error.
