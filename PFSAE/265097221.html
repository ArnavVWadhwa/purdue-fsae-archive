Purdue Formula SAE : Harness: Pinout

Created by Paradise, Hunter, last modified on Dec 06, 2023

Background:

The pinout is the backbone of the wiring harness. At a minimum, it tells which wire ends go where; from there, it is up to the creator to add whatever functionality maximizes their productivity in planning and manufacturing. Whether or not fancy harness CAD software will be used, the beginnings of a harness should always be rooted in a simple paper or Excel pinout. There, thoughts can be planned out and the basics can be learned before diving into complex software and implementation. Of course with enough experience certain stages of harness design can be skipped, but on this team, it would be rare for the same person to design the harness more than once, and it is still highly recommended to have the basic pinout for quick reference in locations where harness CAD might be too slow or inaccessible.

First Steps:

Most commercial electronic components that exist in the world have datasheets, and most of these datasheets contain all the information that one would ever need to use said component. The most important piece of information that comes from these datasheets when trying to design a harness is their pinouts. In order to plan out where wires within a harness are going to go, one must first know the destinations that exist and their purposes. Then taking known vehicular requirements (see Harness: Requirements) such as fuel injection or ignition, routes can slowly be built up and eventually most pins will be filled. Every component should have a pinout somewhere, if it does not, that is probably because its operation is so rudimentary it does not need one, these components are typically just two wire power components, the wiring for which can often be checked using the previous year's pinout. The main module pinouts will be the most important as they will be the limiting factor for most design decisions.

Basics:

For the most part creating the pinout is as easy as choosing a component, looking at its datasheet, and matching the given pins up to the corresponding inputs and outputs on the ECU, PDM, or Dash. This can be gone about in whatever manner is deemed easiest by the designer however it is recommended that this is done in order of signal priority. This means creating the pinout for level 1 components first then level 2 and so on (see Signal Priority below). This can help ensure that higher priority items get first call for space at their desired destinations (see Considerations below). 


Step 1: Locate Component Datasheet

For this example we will be using a RaceGrade M 16-4360 Manifold Absolute Pressure and Temperature Sensor (https://www.milspecwiring.com/1-BAR-MAPMAT-SENSOR_p_2636.html)


Step 2: Search Datasheet for Component Pinout

In this case, multiple different views of the pinout are given. There is the plain text version and also a version depicted by an image. In most cases this graphic representation will not be there and there will be only a table with pin number and pin type.


Step 3: Determine Sensor I/O Type

Please See: Harness: I/O Types

We can tell by the sensor name that we are dealing with a 2-in-1 pressure and temperature sensor. This does not mean that one input carries two sensor values (though this is possible search up flex fuel sensors), it means that one sensor "body", so to speak, houses two sensors within it that share a common reference voltage. Temperature sensors will almost always go to an analog temperature input and pressure sensors will almost always be analog voltage. Learning what types of signals belong to which I/O will take a little bit of time. 


Step 4: Locate Component Pin Destinations

We know where the wires end on the component side, and we know what types of signals they carry, all that is left is to figure out where the other side goes. Taking Signal Priority and Considerations into account, the MAPT is a level 1 sensor so it gets top priority on location. Since its main function is critical to operation and tune of the engine we know it should go to the ECU. The M130 has 8 analog voltage inputs and 4 analog temperature inputs; there should be no problem accommodating all critical sensors with this number of inputs so there should be space for this sensor. It really does not matter which analog voltage and analog temperature pins are used but it can be beneficial to use the same ones that were use the year prior to reduce the amount of things that need to be altered in the vehicle firmware later on. In this case we will be using analog voltage input 1 and analog temperature input 1. 


These are pin 14 on connector A and pin 3 on connector B respectively. A and B are used to denote the two different connectors on the ECU and PDM. A is the larger of the two with 34 pins while B has 26. 

Important Note

 When using analog temperature inputs on the ECU it is important to consider which reference voltage they are being supplied. AT pins 1 and 3 have a 1k pull up to 5V A while AT pins 2 and 4 have a 1k pull up to 5V B.


Speaking of supply and reference voltages, this sensor has not yet been designated any. Looking back at the sensor pinout this sensor requires a 5V supply and 0V reference. The 5V and 0V are both necessary for the pressure sensor while the temperature sensor only requires the 0V reference. Considerations tells us that it is ideal to separate analog and digital supply and reference voltages to minimize noise. This being the first sensor in this example this is not yet a concern. Once this is placed into the pinout though, the analog "line" will be whichever this one is supplied. This noise is less of a concern with the temperature sensor crossing that is inevitable because of the above note since the voltage division will happen internal to the AT pin and thermistors tend to adjust slowly relative to a potentiometer or pressure sensor. In this case analog signals will be designated the "A" supply and reference lines, while "B" will be reserved for digital signals such as crankshaft position and camshaft position. The mapping of splice points is beyond the scope of what is possible at this stage of design. What matters most now is that the signal "to and from" are documented not necessarily the exact path those signals will follow. Saying that pin 1 of this sensor will go to pin 15 of the B connector and pin 3 on the sensor will go to pin 2 of the A connector will suffice for now.


Step 5: Document Connection

After all of the above is determined, or throughout that process, each of those pieces of information should be recorded in a way that is easy to reference now, will be easy to reference later, and contains all the information necessary but not so much that the documentation becomes a mess. There are going to be a lot of connections on whatever is used to document this process, in this case there is such a thing as too much documentation and such will be a consistent struggle as you move through the harness design process. Signal Priority

How important a sensor or component is in the successful operation of the vehicle plays a large role in its placement in the harness. Sometimes decisions must be made to put a certain component over CAN or not to include a certain component at all. Those decisions should be primarily based on signal priority.

  • Highest level components should go to the module that will directly use their signals
  • Low level components should go to the dash or expander
  • Component level determines CAN bus messaging priority. Higher level components will win arbitration.


Wiring Criticality Definitions

Level 1

    • Component is necessary to finish endurance, no matter how long it takes.

Level 2

    • Component is necessary to finish endurance near pace. All basic car functions are operational to do consistent quick laps.

Level 3

    • Component will disable advanced vehicle control features under failure.

Level 4

    • Component is only used for data analysis purposes.

Predicting Pinouts

In order to get ahead on harness design the pinout must be created before many components are selected. In some cases it is possible to select components after the initial pinout is created and adjust it later to suit them (This option is recommended if starting very early or with little prior knowledge). The best way to go about this is to reference the previous years pinout. For a given component the selection of wires it is supplied should not change. The only thing that should will be their arrangement in the connector and possible their wire gauge. Assume that the same component will be used as the previous year for a given application (Design for Inventory) and for anything that requires new components create a generic pinout based on the sensor type. Analog sensors are known to be 3-wire with a 5V, 0V, and signal for example (as learned in the example above). Also make sure early on that any component being considered for the vehicle is in the pinout. It is much easier to take things out later on than it is to add them. 

Considerations

The following are a list of considerations that should be taken account in addition to those indirectly mentioned throughout this guide when designing a wiring harness.

  • Utilize as much of the previous year's pinout to reduce coding load later on
  • Separate analog and digital reference voltages as much as possible to reduce noise
  • Temperature sensors only have a pullup resistor to certain reference voltages depending on the AT pin
  • Certain vehicle specific components have unorthodox pinouts (Ex. CRNK)
  • Crank position sensor has to be UDIG1

Timeline:

  • Summer → Preliminary pinout and component selection complete + master list created
  • Design Season → Edit preliminary pinout based on team changes
  • Post Design → Hopefully there are no mistakes but if a one is noticed post-design fix it ASAP, a pinout being wrong is not an optimization issue the harness will simply not work.
