RE: Kumar, Akash Krishna 


Problem:

Manual pulling of data takes time away from the testing season.


Requirements/Goals:

VehicleDAQ
Maximize Competition Points
  • Maximize testing time
  • Streamlined collection system
  • Special communication tools utilized (if live DAQ)
  • Minimize weight (minimal effect on performance when removed)
  • Must be inexpensive compared to MoTeC alternative
Driver Safety
  • Must be securely placed on the car
  • Mounting should be low
System Reliability
  • Must give an indication that data has been logged
  • Must have a fall-back of manual data pull
    • Switching back and forth between wireless and manual
    • Having both available
System Serviceability
  • Must be easily accessible
Driver Confidence
  • Could have switch/button on the dash to pull data
    • And/or indicator on the dash


Options:

  • Connect ecu to a small wireless router and pull data using wifi 
    • Has limited range but is relatively cheap, a router would cost somewhere between $30-60
  • Use Xbee radio modules and a raspberry pi to transmit data live
    • Tutorial to interface with raspberry pi using Xbee modules: https://circuitdigest.com/microcontroller-projects/raspberry-pi-xbee-module-interfacing 
    • Each Xbee module is around $30-35, RPi will be somewhere between $50-100
