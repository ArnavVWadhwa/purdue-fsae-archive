Chassis Part Structure

As a team, we've come a long way in terms of CAD part structures. It's nearly impossible to decipher the part structure from the 2019 Technical folder. These were the dark times.

Since then, we've standardized a naming convention that makes looking for parts in a flat file structure far easier. Quickly, the format for Chassis is PFWW_CHA_XYZZ where WW = competition year date (e.g. PF22 for 2022), X = Core Assembly, Y = Subassembly, ZZ = Part within Subassembly.

Still, we have a long way to go in terms of making our CAD concise and accessible. Below is a table that includes a non-comprehensive view of the chassis assembly structure by the end of the year.

Enable the Developer tab to run NX Open scripts.

In the Developer tab, click Edit and open the SusPointsImportNX1926.vb script

Edit the file path for the suspoints spreadsheet.

Run the script once. It will take a while to run, go make a coffee. Once the script has been run, do not attempt to run it again. If you need to update it, check the Suspoint VB Script (NX 1926) page to learn how to do that. Before anything else group all FL/FR/RL/RR points into feature groups as shown below. This improves the ease of use of anyone else who comes across this file.

Skeleton Part

This part should contain two sketches on the XZ plane that drive the rest of chassis design. The first sketch places the bulkhead, steering wheel, headrest, main hoop, engine location, and specifies pedalbox adjustment. This is where we make tradeoffs with suspension (CG location targets), aero (Bulkhead size, height), and our ergonomics by shifting things (e.g. driver CG) around. Take in the ergo feedback found when Defining Chassis Design Objectives. Re-read section F.5 of the rules. This is where we use the lap sim to estimate point gains. Most importantly, document these design decisions.

Suspension Links

We start by making a simplified suspension within the wireframe part. Make the wireframe part the work part. Initially, we cannot see the point / curve geometry from other parts (e.g. SusPoints). To view those points, change the SusPoints Reference Set from MODEL to Entire Part.

Now, we can see the points that exist in the SusPoints file. WAVE link the points into the Wireframe part (we can select all the points at once by left-clicking and dragging a box that encloses all the points. Now, add all the points that appear in the part tree to a Feature Group so we avoid cluttering this file. From this feature group, we can group the FR/FL/RR/RL points together in sub-groups as shown. Note: we cannot combine existing feature groups to one larger feature group. E.g. we can't make the FR/FL/RR/RL groups initially, then group them into a higher-level group.

Create lines between the proper points. This is done with the Line feature within the Curve tab.

Creating Chassis Points

Create Chassis Nodes using the Point on Curve/Edge tool within the Point Dialogue.

Assign the Curve Length to an expression (chassisNodeClearance) so it's easier to adjust if needed.

The only nodes that need to be created are the diff tube nodes along with the corresponding 'stub' tube node. In PF22, the diff tube node was placed as an offset point from the RSUS UPR AFT node. The stub node was offset 2 inches in the +X direction. This is shown by the red arrows below. A better way to do this could be to find the resultant force vector at the rocker, place the stub node so the tab is solely in compression (not in bending) and drive the placement of the diff tube off of the stub node. This approach is shown by the dashed orange arrows.

Creating the Wireframe

Reference Geometry

Start by linking any relevant reference geometry from the skeleton part. This includes the Upper SIS height point, BH placement line, MRH placement line, and line from the top of the MRH to the top of the FRH. Make a new feature group of these in the part tree.

Bulkhead

Place a Datum CSYS on the midpoint of the BH placement line. Create a sketch on this CSYS. Use the FSUS chassis nodes and geometric constraints to define the BH centerlines.


Main Roll Hoop

Define a CSYS and sketch at the base of the linked line for the MRH. Define the bend radius as a nominal size according to VR3 nominal bend radii & minimum straight length (link to their bend guidelines here) and according to the rules (regulation F.5.2.1). Make points for the main hoop braces according to regulation F.5.9.4. Place the max width (where the Upper Side Impact Member is) of the MRH 280mm above the lowest centerline point of the Lower Side Impact Member according to regulation F.6.4.4b. Extend the centerline of the MRH on either side ~1inch so the lower horizontal tube gets coped instead of making a miter joint (easier for welding). Additionally, make points for the shoulder harness bar based on ergo feedback.

Front Roll Hoop

Create a point at the front end of the MRH-FRH reference line. This denotes the top node of the FRH. Name the point accordingly. Using this existing point, create two offset points that will drive the 'shoulders' of the FRH. Drive the offset option with expressions, so we can manipulate the FRH form better later on.

Now, create datum planes for the left and right sides of the FRH. These are created via the Three Points subtype and selecting the aft FSUS chassis nodes and the offset point we've just created.

Create a sketch on this plane with the origin as one of the points used to define the datum. Use the 'Specify CSYS' dialog to place the CSYS for the sketch precisely. Create a line from the lower node to just outside of the middle node. From this endpoint, create another line to the offset node created earlier. Create a fillet between these curves according the the same constraints as the bend in the MRH. Be sure to drive these bend radii with expressions. Add a 'point on curve' constraint to keep the node on the midpoint of arc. Extend the bottom curve by 1in for the same reason as we extended the MRH. Follow the same procedure on the other half of the FRH.

Create a datum plane for the top section of the FRH. This comprises of the offset nodes and the top node.

Similar to the sides of the FRH, create a sketch and curve accounting for the bend radius of the top section of the FRH.

Now, we need to make radii that connect the three sketches together. Start by creating datum planes from two lines as shown below for each side of the FRH.

Create a sketch on this datum plane, project the lines that defined it onto the sketch, and add the radius in the form of an arc of the appropriate radius. Don't use the fillet tool here so the arc will update accordingly if the existing lines change.

Use the Trim Curve tool to trim the sketch curves at the shoulder arcs as shown. Remember to create a feature group for the FRH following this operation.

Additional Tubes / Considerations

At this point, creating the rest of the wireframe should be relatively straightforward. Use the line tool and organize elements in Feature Groups (e.g. Front Tubes / Middle Tubes / Rear Tubes).

  • Look into making two traditionally coped & welded tubes one bent tube. Examples of tubes that may be condensed are shown below.

  • Be careful with what is selected when creating new geometry. When making lines, err on the side of selecting the oldest existing feature (e.g. a chassis node / point) instead of something created later on (an endpoint). This improves model stability.
  • Using Feature Curves as a selection rule also improves model stability as shown below.
