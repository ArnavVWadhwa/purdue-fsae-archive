Purdue Formula SAE : Yaw Sensitivity Validation

Created by Zhang, Chenkai, last modified on Jul 05, 2023

Been thinking about yaw sensitivity a lot tonight because I got tired of designing for yaw 15...so throwing this idea on here for people to look at in testing season.

Step 1: Geometry

Prepare at least two sets but preferably more geometries. One set is the base geometry that runs on PF24, and others increasingly messes up yaw sensitivity (for example strakes on the top surface of front wing).

How it mess up yaw sensitivity is a decision you have to make. For example, if have one package that's good for 0-5 and slowly drop off for 5-10, another package that's good for 0-12, but jumps off a cliff for 12+, and another package that's increasing from 0-8 and decreasing 8-15, you will get great data but it's hard to work with since so many things are changing at a time. You will probably have to run more tests for it to be useful. But if you have a package that has its cl decrease by 2% per 5 deg of yaw, another decrease by 4% per 5 deg of yaw, you gain limited insights. But the data will be so much easier to work with. The takeaway here is you will have to strike and balance.

Know how these sets of geometry behave. So that means take these to wind tunnel and do detailed yaw sweeps. Obtain scaling from WT to CFD. Use the scaling to scale on track BC CFD.

Step 2: Testing

Pick a really windy but not gusty day. Get an anemometer or something to keep track of wind speed and direction precisely (obviously very important).

Set up the biggest understeer gradient possible at Grissom and drive as fast as possible in steady state around the circle.

Run without aero and time it. This can be approximated as friction coefficient of tires.

Run with each set of aero and record wind speed and direction in each run. If wind speed and direction changes drastically during the run, rerun until useful data is collected. Time the runs.

Step 3: Validation

Go into the yaw sensitivity sim that I made (or at hopefully made lol), make a the US gradient circle and match the condition and aero package performance. Non-aero needed to obtain the friction coefficient to put in the sim. Mess around with correlation coefficients until the times match up well.
