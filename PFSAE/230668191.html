Purdue Formula SAE : Step 1: MoTeC Software Development

Created by Skrzyszowski, Mateusz, last modified by Paradise, Hunter on Nov 19, 2023

In this step, we will be working in M1 Build. This is where all firmware development for the M130 ECU lives.


Reading
  • M1 Development Manual: M1 Development Manual.pdf
    • This is all about the M1 ecosystem and how each software interacts with each other. This will be incredibly useful to know when approaching the project. However, this text is very dense when it comes to firmware programming, so skip sections as you wish, but beware of important information you may miss as a result.


Required Software

  • M1 Build and M1 Tune: https://moteconline.motec.com.au/ResourceLibrary/Downloads
    • M1 Build is for writing MoTeC firmware, and M1 Tune is for editing parameters created in M1 Build
    • NOTE: The version of M1 Build you will need is 1.4.0.239


Project Creation

  1. You will need to install a development license to be able to compile your code in Build, so you will do this first. Message your lead for the file, and they will send it to you if they haven't already. You just need to double-click on this file and it should install the license.
  2. In M1 Build, click on "Create a new Project". Then select "As an Empty Project" and hit Next. Name this file with the format "purdueAccountUsername_lastName_FirstName LED Onboarding". In Version Name put "2023".
  3. You may see a list of errors pop up when you create the file, just click close on it.

           

Project Settings

  1. Next, you will fill out the Settings for your file.
  2. Open the dropdown in the "Hardware" section and select "M130". This is where you select the ECU you are using if you were to use anything other than an M130.
  3. Navigate to the development license dropdown, and select the development license you installed earlier.
  4. Now you need to fill out the author section. If you do not fill out every box, the project will not compile. Feel free to fill in anything you want here aside from the name section, in which you should put your first and last name.

         


Building the Model

  • The bulk of the firmware will be built under the "Objects" tab. Your objective is to control an LED with 5 different selectable modes in M1 Tune. The 5 different modes are as follows: on at all times, square wave (on for one second, off for one second, repeat), triangle wave (slowly increasing brightness then slowly decreasing), sine wave (slowly increasing brightness then decreasing nonlinearly), and sawtooth wave (slowly increasing brightness until it reaches 100%, then back to 0%).
  • This information is purposefully vague, as the goal of the onboarding is for you to learn the software for yourself and not have your hand held through the project. However, feel free to reach out to your lead with any and all questions regarding this project. Asking questions is ENCOURAGED so please send as many questions as you want.
