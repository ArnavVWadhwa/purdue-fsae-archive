Setting Up Git

First make sure you have a GitHub account and are on the repository 

Open up scholar on Remote Desktop and then open GitHub on a web browser

Go into the fsaeSTAR repository on the browser and copy the URL 

Open up a terminal command, change directory into where you want the git folder to be and then use git clone <paste url>

Basic Git Commands for Linux: https://confluence.atlassian.com/bitbucketserver/basic-git-commands-776639767.html

You are required to setup a personal access token (PAT). Google how to do that. Note you can't view your PAT after creating it, so find a way to preserve it.

If you don't have to type in PAT every time, you can store the PAT in scholar. There are plenty of tutorial on StackOverflow.

Running Sims 

Make sure linuxConfig file is setup in the Python folder of the repository

Change directory into fsaeSTAR/Python 

Run python3 folderBuilder.py

Queueing Multiple Jobs with Dependencies: python3 folderBuilder.py <list of job ids separated by spaces> 

Use squeue to ensure that the jobs have been queued

  • Add -u userID to view only your job
  • Do watch -n 1 squeue to have automatically updating queue list 

If you need to cancel a job then use scancel <job id> 

  • Add -u userID to cancel all your jobs

Using STAR-CCM on Remote Desktop w/ Nodes

Allocating GPU Nodes: salloc --exclusive -A gpu -t 240 -N 1 --constraint=G

  • constraint can either be G or H
  • H nodes are much larger, no need to get it if you can get G nodes

Allocating Scholar Nodes: salloc -A scholar -N 1 -t 240 --exclusive --constraint=B

  • constraint can either be A or B
  • A contains 128 core and B contains 64 cores
  • Get B nodes whenever you can
  • People likes to get A node only use a few core, don't be that guy
    • A nodes are also much more in demand. Don't use it when you don't have to
