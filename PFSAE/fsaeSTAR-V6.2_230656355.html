Post Processing

  • Being revamped to be faster, more accurate, and more comparability.
  • 3D scenes
    • 3D scenes will no longer be exported as PNGs
      • You can't rotate PNG, which make you lose out on information not shown in standard views
      • Contour scenes are pretty useless in static PNGs
      • You can't dynamically change color bar in a PNG
    • STAR Viewer
      • Light weight license free tool
      • You can run this on a potato computer
      • Need scn files, which are very fast to export
        • So in the future, we can just throw 3D scenes in there
    • New 3D scene: vortex breakdown
      • Consists of vortex breakdown contour and surface pressure mapping 
      • New field function: ${WallDistance} < 0.003 || ${Qcriterion} > 100000 ? 10 : ${TotalPressureCoefficient}
        • Filtered CpT 
        • If wall distance is less than 0.003, set total pressure coefficient to a very large number (10), essentially filtering out low total pressure region in boundary layer
        • If Q-criterion is greater than 100,000, filter out total pressure coefficient
          • Meaning if the magnitude of vorticity is 100,000 greater than strain rate, we want to filter it out
          • If we don't do this, high vorticity area will so show up as "vortex breakdown" which is counterproductive
      • Contour is set to -0.25
  • 2D Scenes
    • New scenes:
      • Velocity X: velocity in x direction. Good for showing this like vortex breakdown
      • Velocity Z: velocity in z direction. Important for identifying local angle of attack
      • Inwash: shows inwash/outwash magnitude. Extremely important for undertray design
      • New field function: -$${NormalizedVel}[1] * ($${Position}[1]/abs($${Position}[1]))
    • Moved velocity glyph to additional design scenes
      • You can't do image delta with velocity glyph, so it's much less useful in the ages of SimComparisonTool
      • And velocity glpyhs are horrendously slow to export and unpack
  • Other changes
    • Restructured scenes organization in STAR
      • Eliminated the need to hardcode displayer names into fsaeSTAR
  • Added length wise local load plot
    • This shows where downforce is coming from better than simple, per-part based system
