Purdue Formula SAE : PF20 Frame Sensitivity Testing

Created by Rumsey, Helen Nicole, last modified on Nov 25, 2022

Goal: To connect drivability and frame geometry. 

Procedure: (Spring configuration 1: 300 F, 400 R & Spring configuration 2: 450 F, 400 R)

Slaloms in baseline configuration and final configuration.

Per Configuration:

  1. On-track torsion test of configuration

  2. Spring Configuration 1: skidpad, understeer gradient, slaloms until driver feels consistent

  3. Spring Configuration 2: skidpad, understeer gradient, slaloms until driver feels consistent

  4. Post process data using on-track Motec channels

  5. Check times for consistency & ask drivers for drivability feedback. If the car feels “undrivable” make ARB changes until drivable again. If ARB changes don’t fix “undrivability” problem, minimum tuneability has been reached.

  6. Remove chassis tubes and repeat.

Final configuration egress (when the nSIS tubes are removed). 

On-track Torsion Test: With the driver in the car, at the same tire pressure and spring for each test, put a 2x4 under the front left tire and record the shock displacement.

Analysis Location: mefsae\2023\4_Chassis\3_Simulation\PF20_Frame - located in this folder is the ANSYS files used to analyze the frame before the test and determine removal order, I2 data from testing, and calculators used to post process the data

Analysis:

  • Compare shock values to ANSYS model for each frame configuration (likely use skidpad and determine how many Gs were pulled)
  • Use shock data to determine sensitivity value (and compare to expected sensitivity value)
  • Compare braking and acceleration to ANSYS model (is there a large enough effect on LLTD for this to matter?) 
  • Use driver feedback to determine an appropriate sensitivity range (how low can we go before the car gets hard to drive/ is it still drivable at rules minimum?)

Open Concerns/ Questions:

  • To remove a tube do we need to cut the whole tube out or just slice it to negate it's effect?
  • What's mounted to the tubes we want to remove that would cause issues?

Results & Concluding Thoughts:

The frame in its final configuration is still tunable to reach the same LLTD desired ranges (but at the cost of spring configurations that might not be as desirable for other suspension characteristics).  Drivers still achieved the same skidpad times, but less comfortable (which could be due to tube removal or a lot of other factors were changing as the day went on like tire temperature, track temperature, engine temperature, ect.). 

As seen by the initial analysis (but not confirmed by the data since some of the data was lost), the first 3 tubes to be removed had little impact of sensitivity to LLTD. We didn't try to tie the on track data back to FEA analysis on sensitivity since the margins on track are smaller than I would feel comfortable saying the model could predict. 

The last tube removed (upper nonstructural SIS) had the largest impact on sensitivity to LLTD in FEA and I would deem as the big tube to consider if it should be removed or not. We didn't have data to compare apples to apples suspension configurations between CHA1 and CHA5 but we were able to achieve the same range of LLTD values. Note that later on in the design process we added this tube back to the frame design in order to help with a front impact (refer to PF23 frame analysis for more information.
