Why do we need this?

The purpose of this page is to provide CAD documentation standards. CAD documentation is important to track the parts of the car as we are manufacturing and testing the car. In addition, we consider CAD to be the single source of truth when building the car, so standardizing file names in an organized and structured way it beneficial to all parties.

General

Assemblies should be named with plain English, no part number necessary. We'll be utilizing pascal case for these. Examples of pascal case are below. 


General rules:

  • Avoid abbreviations
    • People won't be confused and there is 0.0 chance of miscommunication
  • Keep it short and sweet
    • Words like "The", "Assembly", etc. should be omitted
  • Go as deep in the tree as you need
    • This is one of the reasons that we just name our assemblies in plain English – knowing an assemblies parent is now a question of just knowing the car.

Is this a physical, singular part? Follow this route.

Example part

Some correct examples below:

  • The upright as is machined
  • The brake pedal as is machined
  • The standalone, individual suspension tube
  • The brake light tab, as is from the press brake

Some incorrect examples below: 

  • The entire suspension upper control arm
  • The suspension upright, with bearings pressed in
  • The frame

For returning / more experienced members, you can also ask yourself: is this a singular wave linked part? 

Note that you should not have multiple bodies linked into 1 file. 

This part should have a part number associated with it. This is done because of a few differences between assemblies and parts. 

  • Parts have drawing, for our purposes assemblies don't
  • Revisions of parts are possible as old part numbers can be thrown away without consequence.  This prevents "Pushrod1" ... "Pushrod2" ... "PushrodFINAL" ... "PushrodFINALFINAL" ... "PushrodENDGAME" ... etc.
  • Parts need to be meticulously tracked in manufacturing - at design freeze, the final part numbers that are in the car are the ones to be tracked in the clear to build spreadsheet (more on this as we get closer to manufacturing season)

Specifics:

*IMPORTANT AND NEW THIS YEAR*

Will this item be driving on the competition car?

  • 1 = YES
  • 0 = NO
  • We will use a 4 digit part number. This allows for, throughout the season, 9999 total parts to be created, per sub-team. I hope this team never reaches the stage where we revise through that many parts.
  • Team leads to define and organize their part numbers below as they see fit. In my opinion, this doesn't really need a structure since the assemblies are written in plain english - as long as you document the part number and organize it in you're team center folders, you're good.

Revisions: 

Revisions can be handled through Teamcenter. 

Revisions should only be used if you do not care which part is being used from your revision. If you need to use your new part that you are designing, IT DESERVES A NEW PART NUMBER!

Example:

  1. You want to make a fillet change on an upright for DFM far into the manufacturing season. Both parts have positive margin and the fillet change doesn't cause any new collisions with mating parts. This would be a revision. 
  2. You need to change the lower control arm suspension point due to an error in our collision analysis. This is a new part number and the old part number gets deleted from the main PF24 assembly.

Revisions and backups should be saved using the "Save a copy tool" with a letter added to the end of the part name in alphabetical order.  (see: https://wiki.itap.purdue.edu/x/po2RC for more detail)
