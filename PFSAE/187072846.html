Purdue Formula SAE : PF22 Aero Part Tracking/CAD Standards

Created by Unknown User (green338), last modified on Jul 22, 2021

All layout files, assembly parents, and linked files should be saved in their subfolder in Z:\2022\10_CAD-22\4_Aerodynamics

Layout files should be named "PF22_AER_(part code)_Layout (example PF22_AER_RW_Layout). Assembly parent files should be named PF22_AER_(part code)_Assembly_Full (example PF22_AER_FW_Assembly_Full). Both of these files should be under a parent file named PF22_AER_(part num)_(part code)

PartIncludesPart CodePart Number
Reference GeometryAll reference geometry needed from other subsystemsRefGeo0000
Front WingFront wing aero surfaces, mounting, tabs, and tierodsFW1000
Side Wing/UndertraySide wing aero surfaces, undertray aero surfaces, mounting, tabs, tierods, fan shroudsSW_UT2000
Rear WingRear wing aero surfaces, mounting, tabs, tierods, and DRS hardwareRW3000
BodyworkBodywork and nosecone aero surfaces, mountingBW4000

Any part files that are under these parents should be stored in easy to identify subfolders in each part folder. The only part files in the FW, RW, SW, etc subfolders in Z:\2022\10_CAD-22\4_Aerodynamics should be the top level subpart file, the layout file, and assembly file. Observe:

There should also be no part files in the top level aero folder aside from the PF22_AER.prt assembly and global aero part files (tierods, bolts, machined parts specific to our subteam). Remove any files you accidentally save there.

When linking bodies, DO NOT link an entire subsystem assembly, or any bodies. the best way to do this is to bring in faces you need. Bodies will come with a default mass and cause issues down the line.
