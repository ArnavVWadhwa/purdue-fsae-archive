Purdue Formula SAE : Suspension Feedback

Created by Unknown User (reyno105), last modified on Apr 11, 2021

Judge: Keith Wittmer


  • Weight distribution:
    • Picked up on weight distribution sweep having issue of not accounting for adjusting suspension balance based on weight distribution
    • Provided recommendation for analyzing:
      • for each weight distribution adjust suspension rates to maintain the same oversteer / understeer balance. Metric could be front to rear slip angle ratio or steering input. (Keith didn't mention understeer gradient specifically, but that's a thought i had from this)
  • For simple events in lap sim, should use a transient sim to compare. This is only really for accel. Run a transient accel event and see if it's much off of lap time output. The higher fidelity model may have important behaviors being missed.
  • Tires:
    • I forgot to mention camber sensitivity and got called out for that
    • Should discuss pressure sensitivity
    • Should be careful to not show beyond the data in model because it looks wrong
  • Rates:
    • recommended to make it clearer that we have adjustability, should discuss how much adjustability we use
    • Improve discussion and thinking of design for aero. Need to be clear it's more than not hitting it into the ground. (should give a value on FW height that cause big loss. should say we aren't very roll sensitive)
    • In general, keith made point that a motorsports background judge is going to be very up for hearing suspension design to support aero
    • Discussion of fact that higher spring rates harms adjustability (tire rates drive)
  • Keeping rates down while accomodating aero:
    • Anti dive:
      • Discussion led to fact that we should probably use anti-dive. It's hard to decide the most we could get away with. It drives higher suspension loads and maybe makes packaging hard, but couldnt come up with a limit outside of this. But some arbitrary amount that's clearly not going to nuke anything, like 15%, probably makes sense.
    • Bump stops:
      • If you have one event that really drives needing bigger springs, could engineer bumpstops to handle that event so that you can run softer springs for other maneuvers (and still not hit the bumpstop)
      • IE, bumpstop for braking case that doesnt get used in any other case
    • Progressive rates: agreed these are bad because of balance harm corner entry
  • Discuss our tuning program:
    • Need to take credit for what we dide - run skidpad, set ARBs. Run big skidpad, set aero.
    • HOPEFULLY more lap sim correllation
  • Sim software:
    • was surprised we use adams because of how much a pain it is. Mentioned dymola and car sim. (Several judges have pointed out car sim as being more intuitive at this point)
    • Encouraged all steps to pass along lap sim tool
